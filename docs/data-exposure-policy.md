## 1. 4등급 분류 (모든 UI 요소는 이 중 하나로 분류돼야 함)

| 등급 | 정의 | 예시 | 공개 UI |
|---|---|---|---|
| **A. 자체 가공 신호** | 우리가 시계열·변화율·감지 로직으로 만든 결과 | "7일 내 최저가 경신 -12%", "랭킹 ↑8" | ✅ 무제한 |
| **B. 단일 소스 사실 인용** | 평점·리뷰수 등 단건 수치, 1~2개만, 출처 표기 필수 | "화해 평점 4.5 · 567 리뷰 [↗]" | ✅ 개별 상품 상세만 |
| **C. 외부 원본 표·리스트** | 소스의 랭킹·카테고리 트리·어워드 리스트 그대로 | "화해 스킨케어 TOP 20" 복제 | 🔴 **금지** |
| **D. 외부 저작물** | 이미지 핫링크·리뷰 본문·상세 설명 복제 | `img.hwahae.co.kr` 참조, 리뷰 원문 | 🔴 **금지** (공식 제휴 CDN 제외) |

## 3. UI 컴포넌트 게이트

모든 컴포넌트는 렌더하는 데이터가 **A 또는 B** 등급인지 PR 리뷰에서 확인.

### ✅ 현재 안전 컴포넌트

| 컴포넌트 | 등급 | 근거 |
|---|---|---|
| `PriceChangeCard` | A | 자체 계산 `dropRate`, `rankDelta` |
| `PriceChart` | A | 자체 시계열 스냅샷 시각화 |
| 홈 "오늘의 가격 변동" 섹션 | A | 자체 가공 랭킹 |
| 쿠팡 컬렉션 최저가 | A+affiliate | 자체 계산 + 파트너스 딥링크 |
| 제품 상세 페이지 | A | 자체 스냅샷 차트 |

### ❌ 현재 누락·위반 가능 패턴 (도입 전 검토)

- "화해 스킨케어 TOP 20" 섹션 → **금지 (C)**
- "올영 어워즈 수상작 리스트" 섹션 → **금지 (C)**
- 화해·글로우픽 이미지 직접 표시 → **금지 (D)**
- 리뷰 본문 노출 → **금지 (D)**

## 4. 기술 체크리스트

| 항목 | 구현 상태 |
|---|---|
| 크롤 User-Agent 식별 | ⏸ 후속 (`hotinbeauty-bot/1.0 (contact@…)`) |
| robots.txt 자동 점검 | ⏸ 후속 |
| 요청 속도 제한 (소스당 2초 간격) | ✅ `coupang-search.ts` 랜덤 지연 반영 |
| 이미지 파일 저장 금지 | ✅ URL만 메타로 |
| 리뷰 본문 저장 금지 | ✅ 화해는 평점·리뷰수만 |
| 외부 링크 `rel="noopener"` | ✅ 제품 상세 CTA |
| 단건 인용 출처 링크 의무 | ⏸ ExternalRef 컴포넌트 예정 |
| Cease-and-desist 대응 플레이북 | ⏸ runbook 작성 필요 |

## 5. 추가 개발 시 체크리스트

신규 UI 컴포넌트 PR 템플릿에 포함:

```markdown
## 데이터 노출 분류
- [ ] 이 컴포넌트가 표시하는 모든 데이터는 A(자체 신호) 또는 B(단건 인용+출처) 등급이다
- [ ] C(외부 원본 리스트·표 재현) 해당 없음
- [ ] D(이미지 핫링크·리뷰 본문) 해당 없음
- [ ] 외부 소스 인용 시 출처 링크와 소스명이 표시된다
- [ ] `docs/data-exposure-policy.md` 매트릭스와 일치한다
```

## 6. Cease-and-Desist 대응 절차 (후속 작성 예정)

1. 접수 즉시 해당 소스의 UI 노출 **전면 중단** (feature flag)
2. 24h 내 관련 캐시·CDN 제거
3. 법무 검토 → 사실관계 회신
4. 필요 시 데이터 삭제·링크 제거
