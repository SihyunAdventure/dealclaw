## 0. Executive Summary (4줄)

1. **화해·글로우픽 데이터의 "전체 랭킹 재구성"은 법적 실행 불가** — TOS 4대 금지조항 + robots.txt 전면 Disallow + 잡코리아 v 사람인 판례(건당 50만원). **단, 개별 사실 인용·링크는 상대적으로 안전한 활용 범주** (야놀자 v 여기어때 판례, 저작권법 §28 인용 조항).
2. **올리브영은 정반대의 기회** — 공식 "쇼핑 큐레이터" 제휴 프로그램(수수료 7%, 쿠팡 3%의 2.3배) 보유. 가입 자체가 합법 경로.
3. **권장 방향: Wedge Pivot (시나리오 B+)** — 쿠팡 + 올영 **듀얼 affiliate** 기반 "가격·랭킹 변화 감지 엔진". 화해·글로우픽은 **내부 분석 + 개별 인용 + 링크** 범위에서만 활용. Phase 6 UI 11h 계획은 폐기하고 **1h 스모크 (`/product/[id]`)로 실측 후 scale**.
4. **"자체 가공 신호 우선" 원칙** — 공개 UI는 우리가 계산한 지표(최저가 경신, 가격 시계열, 변동률)가 주, 외부 소스의 원본 표·리스트는 재현 금지.

## 2. 조사 결과

### 2.0 법적 스펙트럼 — "무엇을 어느 정도"가 핵심

단순한 go/no-go가 아니라 **규모·체계성·목적·결과** 4요소에 따라 합법↔위법 스펙트럼. 판례 두 축:

| 판례 | 결과 | 특징 |
|---|---|---|
| 잡코리아 v 사람인 | 🔴 침해 (건당 50만원 × 396건) | **전체 DB 체계적 복제 + 자사 랭킹에 재사용 + VPN 우회** |
| 야놀자 v 여기어때 | 🟢 무혐의 | **50 필드 중 3~8개만 부분 추출 + 기술 차단 없음** |

**6단계 행위 스펙트럼** (우리 적용 기준):

| # | 행위 | 예시 | 판정 |
|---|---|---|---|
| ① | 링크만 | "글로우픽에서 보기 →" | ✅ 명시 허용 (올영 법적고지도 링크는 OK) |
| ② | 개별 사실 인용 + 출처 | 상품 상세에 "글로우픽 에센스 1위 · 평점 4.47 (출처)" 한 줄 | ✅ 공정이용 (저작권법 §28) |
| ③ | 부분 필드 추출 (소수) | 50 필드 중 3~8개만, 맥락 없음 | ✅ 야놀자 판례 방어선 |
| ④ | 출처 표기 + 부분 목록 + 자체 가공 추가 | "글로우픽 TOP 5 중 쿠팡 최저가 비교" (우리가 계산한 가공 지표 추가) | ⚠️ 회색 — 방어 가능하나 리스크 |
| ⑤ | 체계적 전체 미러링 | 에센스 TOP 100 전체를 매일 크롤해 자사 랭킹 섹션으로 재구성 | 🔴 **잡코리아 판례 = 침해 확정** |
| ⑥ | 이미지 핫링크 / 리뷰 본문 복제 | `img.hwahae.co.kr` 직접 참조, 리뷰 원문 노출 | 🔴 저작권 별개 침해 |

**우리 규칙**: **①②③ 자유 / ④ 소스별 개별 판단 + 법무 검토 / ⑤⑥ 전면 금지.**

> **주의**: 본 문서는 제품·운영 의사결정용 가이드이며 최종 법률 의견이 아니다. 특히 ④ 회색지대는 법무 검토 전 공개 UI에 싣지 않는다.

**판정 (세분화)**:
- 🔴 **금지**: 화해 랭킹을 통째로 우리 사이트에 재구성 (§2.0 ⑤⑥) — Phase 6 원안이 이것. 차단 조건.
- ⚠️ **회색**: 화해 랭킹 부분 목록을 출처 표기하며 자체 가공 추가 (§2.0 ④) — 법무 검토 없이 권장 X.

### 2.2 올리브영 법무 — 🟢 GREEN (제휴 경로 확보 가능)

| 증거 | 내용 |
|---|---|
| 법적고지 | "개인적인 용도로 사용하거나 고객의 컴퓨터에 저장할 수 있으며, **개인적인 홈페이지나 문서로부터 본 서버로의 링크 등은 가능**. 다만 사전 동의 없이 임의로 저장·수정·배포·제공·전송·공유·이용은 저작권 침해" |
| **공식 affiliate** | **쇼핑 큐레이터 프로그램** — CJ 공식 운영 |
| 수수료 | **직접 7% / 간접 3%** (유튜브 쇼핑 파트너로도 등록됨) |
| 핵심 조건 | 공식 발급 페이지에서 생성한 링크만 유효 · 임의 수정 금지 · 성인/도박 사이트 금지 |
| 회색지대 | 큐레이터 = "링크 공유" 목적. 올영 DB를 UI 재노출하는 행위는 **큐레이터 범위 외** |

**해석**: 올영 **내부 신호로 활용 + 공개 UI는 링크+자체 분석**만 노출 시 합법. **제휴 가입 시 자체 분석 서비스 형태도 해석 여지 有** (법무 검토 권장).

### 2.3 쿠팡 홈 지표 — 내부 데이터 필요

**확보된 벤치마크**:
| 지표 | 업계 기준 |
|---|---|
| affiliate CTR | 0.7~1.2% (1% 넘으면 excellent) |
| 뷰티 CVR | 4.55% (개인케어 6.8%) |
| 쿠팡 뷰티 파트너스 수수료 | **최대 3%** |

**수익성 재계산 (뷰티 10만원 주문당)**:
| 플랫폼 | 수수료 | 수익 | 상대 |
|---|---|---|---|
| 쿠팡 파트너스 | 3% | 3,000원 | baseline |
| **올영 큐레이터** | **7%** | **7,000원** | **+133%** |

**함의**: "통합 포털"로 트래픽 분산 시 EPC 희석. 뷰티 단일 세그먼트에서는 **올영 7% > 쿠팡 3%** — 포지션 재정의 강력 근거.

### 4.2 1h 스모크 상세 (Wedge 0)

**구현 범위**:
- 라우트: `/product/[id]` — id는 임의 3개 (쿠팡·올영 모두에 있는 상품 선정)
- 컴포넌트:
  - `<PriceChart>` (이미 `components/price-chart.tsx` 존재, 활용)
  - `<PriceLowBadge>` — 7일/30일 최저가 경신 뱃지
  - `<DualAffiliateButton>` — 올영(주) + 쿠팡(보조)
- API: `GET /api/product/[id]` — 가격 시계열 + 최저가 경신 flag
- GA 이벤트: `page_view`, `price_chart_view`, `affiliate_click_oliveyoung`, `affiliate_click_coupang`
- 구독 CTA: 기존 SES 파이프라인 재사용

**측정 기준 (1주 관찰)**:
| 지표 | kill | continue | scale |
|---|---|---|---|
| 페이지당 CTR | < 0.5% | 0.5~1% | ≥ 1% |
| 전환 (affiliate click → 구매) | 불가측정 | 설문 | 파트너스 대시보드 확인 |
| 체류 시간 | < 15s | 15~30s | ≥ 30s |
| 구독 CTA 전환 | < 0.5% | 0.5~2% | ≥ 2% |

→ 결과에 따라 Wedge 2~6 진행/중단 결정.

### 4.4 공개 UI 운영 원칙

Wedge Pivot의 핵심은 **"무엇을 수집하느냐"보다 "무엇을 공개 UI에 어떻게 노출하느냐"** 에 있다. 모든 후속 UI 작업은 아래 4원칙을 기본값으로 둔다.

1. **홈·피드·컬렉션은 A 등급(자체 가공 신호)만 노출** — 외부 소스 원문 느낌이 나는 요소는 넣지 않는다.
2. **상품 상세만 A+B 허용** — 외부 단건 인용은 상품 상세에서만, 소스당 1줄·총 2개 이하.
3. **공개 정렬 기준은 항상 우리 지표** — "오늘 가격 경신", "최근 7일 하락률" 같이 우리가 계산한 순서만 사용.
4. **raw source 데이터는 내부 도구로 격리** — 원본 랭킹, 크롤 snapshot, 품질 로그는 admin/internal 화면에서만 사용.

### 4.5 수집 데이터 안전 UI 노출 플레이북 ⭐

이미 수집했거나 수집 중인 모든 데이터를 버리지 않고 **§2.0 스펙트럼 ①②③ 범위** 내에서 최대한 활용.

#### 4.5.1 데이터 소스별 허용·금지 매트릭스

| 데이터 소스 | 내부 DB 보관 | 개별 상품 상세 인용 | 변화 감지 신호 | 전체 랭킹 재현 | 이미지 |
|---|---|---|---|---|---|
| **쿠팡** (자체 수집) | ✅ | ✅ | ✅ | ⚠️ 파트너스 약관 확인 | ⚠️ 공식 CDN만 |
| **올영** (공식 큐레이터 가입 후) | ✅ | ✅ | ✅ | ⚠️ 큐레이터 범위 내 | ⚠️ 큐레이터 제공 자원만 |
| **화해** | ✅ (내부만) | ✅ (1~2줄 인용 + 출처) | ⚠️ 자체 계산 값만 | 🔴 금지 | 🔴 금지 |
| **글로우픽** (수집 예정 시) | ✅ (내부만) | ✅ (1~2줄 인용 + 출처) | ⚠️ 자체 계산 값만 | 🔴 금지 | 🔴 금지 |

#### 4.5.2 "자체 가공 신호 우선" 원칙

공개 UI에 표시되는 모든 정보는 다음 분류 기준을 통과해야 함:

| 분류 | 정의 | 예시 | 출처 표기 | UI 노출 |
|---|---|---|---|---|
| **A. 우리가 만든 신호** | 시계열 가공, 변화율, 감지 결과 | "7일 내 최저가 경신", "가격 변동률 -12%", "랭킹 8위 상승" | 불필요 | ✅ 무제한 |
| **B. 단일 소스 사실 인용** | 평점·리뷰수 등 단건 수치 1~2개 | "화해 평점 4.5 · 567 리뷰" | 필수 (소스명 + 링크) | ✅ 개별 페이지만 |
| **C. 외부 원본 표·리스트** | 소스 랭킹·카테고리 트리·어워드 리스트 그대로 | "화해 스킨케어 TOP 20" 복제 | - | 🔴 금지 |
| **D. 외부 저작물** | 이미지·리뷰 본문·상세 설명 | `img.hwahae.co.kr` 참조, 리뷰 복제 | - | 🔴 금지 (공식 제휴 CDN 제외) |

**게이트**: 모든 컴포넌트는 "A 또는 B" 범주만 렌더링하도록 PR review 체크리스트에 명시.

#### 4.5.3 Wedge별 UI 구성안

##### Wedge 0 — `/product/[id]` (스모크)
```
┌─────────────────────────────────────┐
│ [상품명] — 우리 DB 기준              │  ← A
├─────────────────────────────────────┤
│ 📉 가격 시계열 차트 (30일)           │  ← A (자체 계산)
│ 🏷  "7일 내 최저가 경신" 뱃지         │  ← A
│ 💰 현재가 / 최저가 / 변동률           │  ← A
├─────────────────────────────────────┤
│ 평가 요약 (선택, 있는 경우만)         │
│  · 화해 평점 4.5 · 567 리뷰 [↗]     │  ← B (단건, 출처 링크)
│  · 올영 평점 4.3 · 1.2k 리뷰 [↗]    │  ← B
├─────────────────────────────────────┤
│ 🛒 올영에서 구매 (7%)  ← 우선         │  ← 공식 큐레이터 링크
│ 🛒 쿠팡에서 구매 (3%)                 │  ← 공식 파트너스 딥링크
└─────────────────────────────────────┘
```
**위반 없음**: A·B만 사용. C·D 전무.

##### Wedge 2 — `/` 홈 "변화 감지 피드"
```
┌─────────────────────────────────────┐
│ 🔥 오늘 가격 경신 TOP 10              │  ← 우리 랭킹 (자체 가공)
├─────────────────────────────────────┤
│ 1. [상품] -18% 경신 · 쿠팡 / 올영    │  ← A + 소스 꼬리표
│ 2. [상품] 랭킹 +8 상승 · 올영        │  ← A
│ ...                                 │
└─────────────────────────────────────┘
```
**핵심**: "쿠팡 인기 TOP 10" 이나 "올영 베스트" 가 아님. **우리가 계산한 "오늘 가격 경신"** 이라는 자체 지표를 기준으로 정렬한 자체 리스트. C 금지선 우회.

##### Wedge 3 — 상품 상세 확장
Wedge 0 + 요소 추가:
- 📊 랭킹 시계열 (우리 DB의 일별 스냅샷 기반, **모든 소스 공통 포맷**으로 재가공)
- 🔔 "이 상품 가격 경신 알림" SES 구독 버튼

##### Wedge 4 — 크로스소스 비교
```
┌────────────────────────────────┐
│ [상품]                          │
│ 쿠팡 ₩32,000  (최저)             │  ← A (자체 계산 "최저")
│ 올영 ₩35,000                    │  ← A
└────────────────────────────────┘
```
- 원본 랭킹·리스트 재현 X
- "올영 어워즈 수상작" 은 **개별 상품 뱃지로만** ("이 상품은 2025 올영 어워즈 수상작") — 소스 리스트 복제 아님

##### Wedge 5 — 알림
이메일 본문에 **우리 가공 신호 + 공식 링크만**. 외부 랭킹 표 이미지 임베드 금지.

#### 4.5.4 현재 코드베이스 기준 UI 적용 우선순위

| UI surface | 표시 데이터 | 허용 등급 | 구현 메모 |
|---|---|---|---|
| 홈 컬렉션 카드 / 피드 카드 | 현재가, 변동률, 최저가 경신, 소스 chip(쿠팡/올영) | A | 기존 `product-card`, `deal-list-item`, `price-display` 계열 컴포넌트는 **자체 계산 필드만** 주입 |
| 상품 상세 상단 | 가격 차트, 최저가 뱃지, 듀얼 affiliate CTA | A | Wedge 0 핵심. 외부 원본 랭킹 문구 없이도 구매 이유 전달 가능 |
| 상품 상세 하단 참고 정보 | 화해/올영/글로우픽 평점·리뷰수·수상 여부 1~2줄 | B | **소스당 1줄, 페이지당 최대 2개**. 반드시 원문 링크와 함께 노출 |
| 이메일/푸시 알림 | "오늘 -18%", "7일 내 최저가" 같은 트리거 문구 + 공식 링크 | A | 외부 플랫폼의 랭킹 표, 썸네일, 리뷰 본문 미포함 |
| 관리자/내부 대시보드 | 원본 랭킹, raw snapshot, 크롤 품질 로그 | internal only | raw 데이터는 내부에서만 사용. 공개 라우트와 DB view 분리 필요 |

**운영 규칙**:
- 홈/리스트/컬렉션 계열 페이지는 **A 등급만 허용**. 외부 단건 인용(B)도 넣지 않는다.
- **B 등급은 상품 상세 1페이지에 한정**하고, 소스당 1줄·총 2개 이하로 제한한다.
- "올영 어워즈 수상" 같은 정보는 **불리언 배지 1개**로 축약하고, 어워드 전체 리스트 페이지는 만들지 않는다.
- 화해·글로우픽 raw rank는 내부 매칭·품질 평가·알림 우선순위 산정에만 사용하고, 공개 UI 정렬 기준으로 직접 쓰지 않는다.

#### 4.5.5 법적 안전 위한 기술 체크리스트

| 항목 | 구현 |
|---|---|
| 크롤 User-Agent | `hotinbeauty-bot/1.0 (contact@…)` 식별 가능하게 |
| robots.txt 존중 | 자동화된 점검 — 차단 경로 감지 시 해당 플래그 |
| 요청 속도 | 소스당 최소 2초 간격, 동시 1 연결 |
| 저장 범위 | 이미지 파일 저장 금지 (URL만 메타로, 렌더 시점엔 미사용) |
| 리뷰 본문 | 수집·저장 금지. 토픽 태그·평점 숫자만 |
| 공개 UI 필드 allowlist | 컴포넌트별로 "허용 필드" 명시 (DB schema에서 분리) |
| 출처 표기 링크 | 외부 단건 인용 시 `rel="noopener"` + 소스 도메인 링크 |
| Cease-and-desist 대응 플레이북 | 즉시 해당 소스 UI 노출 중단 + 24h 내 제거 (runbook 작성) |

## 7. 장기 비전 (6개월)

**6개월 후 defensible한 것은 UI가 아니라 "의사결정 엔진"** (Codex 원문 인용)

| 레이어 | 6개월 후 모습 |
|---|---|

| 방어력 | 시계열 depth + 교차 비교 알고리즘 + 구독자 리스트 |

- 월 affiliate 수익 < 10만원 → 재설계
- 주간 구독 증가율 < 5% → wedge 재검토
- 어느 소스도 cease-and-desist 수령 → 해당 소스 제거 + 재정의
